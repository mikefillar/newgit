Git Staging

One of the function of git is the staging environment. Staged files that are ready to be commited to the repository you are working on.

Git branch merge

To merge the master and the new branch

  1. git checkout master
  2. git merge new branch
  3. save

Git clone

git clone is a copy copy of the repository including all logs and version of your project git clone "url or link of repository"

Git revert, Reset and Restore

  1. git revert #code use to delete specific commit
  2. git revert HEAD --no-edit use to revert the last commit
  3. git revert HEAD #code use to revert the specific commit using the 7 character #code
  4. git reset #code use to reset specific commit using the 7 character #code
  5. git reset --hard 'HEAD@{1}' used to reset the specific changes to the repository
  6. git restore --staged (filename) to restore deleted file

Git log and reflog

  1. git log used to check activity logs of the repository
  2. git reflog used to check the logs with reference number to verify specific changes to repository
